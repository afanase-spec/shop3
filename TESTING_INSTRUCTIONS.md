## Быстрая проверка (5 минут)

### 2️⃣ Проверка модального окна

**Действие**: Нажмите на кнопку с ценой на странице товара

**Что должно произойти**:
- ✅ Открывается модальное окно по центру экрана
- ✅ Видно название товара
- ✅ Видна цена за единицу
- ✅ Есть кнопки "-" и "+"
- ✅ Есть поле ввода с числом 1
- ✅ Показана итоговая сумма "Итого: 89,90 ₽"

**Тест кнопок +/-**:
- Нажмите "+" → количество стало 2, сумма удвоилась
- Нажмите "-" → количество стало 1, сумма вернулась
- Введите вручную "5" → сумма пересчиталась
- Попробуйте ввести "0" → автоматически стало 1
- Попробуйте ввести "100" → автоматически стало 99

**Тест кнопок модального окна**:
- Нажмите "Отмена" → окно закрылось, товар НЕ добавлен
- Снова откройте окно, нажмите "Добавить в корзину" → окно закрылось, появилось уведомление

---

### 3️⃣ Проверка суммы в header

**До добавления товара**:
- Посмотрите в правый верхний угол
- Если корзина пустая → ничего нет под иконкой корзины

**После добавления товара**:
- Под красным badge с количеством появилась сумма
- Формат: мелкий фиолетовый текст "89,90 ₽"
- Сумма соответствует цене товара × количество

**Проверка обновления суммы**:
1. Добавьте еще один товар → сумма увеличилась
2. Перейдите в корзину → измените количество → вернитесь на главную → сумма обновилась
3. Удалите товар из корзины → сумма уменьшилась
4. Очистите корзину полностью → сумма исчезла

## 🐛 Возможные проблемы и решения

### Проблема 1: Модальное окно не открывается
**Причина**: Bootstrap JS не подключен  
**Решение**: Проверьте, что в header.php есть:
```html
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
```

### Проблема 2: Сумма не отображается в header
**Причина**: База данных не обновлена  
**Решение**: Выполните SQL скрипт из `UPDATE_PRODUCTS_TABLE.sql`

### Проблема 3: Сумма не обновляется после добавления товара
**Причина**: JavaScript ошибка  
**Решение**: 
1. Откройте DevTools (F12)
2. Перейдите на вкладку Console
3. Добавьте товар в корзину
4. Посмотрите ошибки
5. Проверьте, что API возвращает `cartTotal`

### Проблема 4: Неправильный формат суммы
**Причина**: Локаль не поддерживается  
**Решение**: Проверьте в консоли:
```javascript
(1234.56).toLocaleString('ru-RU', {minimumFractionDigits: 2})
// Должно вернуть: "1 234,56"
```

## 🎯 Идеальный сценарий использования

1. **Пользователь открывает товар**
   - Видит чистую страницу без лишних контролов
   
2. **Нажимает кнопку "89,90 ₽ +"**
   - Открывается красивое модальное окно
   
3. **Выбирает количество 3 шт**
   - Нажимает "+" два раза
   - Видит "Итого: 269,70 ₽"
   
4. **Нажимает "Добавить в корзину"**
   - Окно закрывается
   - Появляется зеленое уведомление
   - В header под иконкой корзины появляется "269,70 ₽"
   
5. **Переходит в каталог**
   - Сумма "269,70 ₽" все еще видна в header
   
6. **Добавляет еще товар на 100 ₽**
   - Сумма в header обновляется до "369,70 ₽"
   
7. **Переходит в корзину**
   - Видит оба товара
   - Общая сумма совпадает с той, что была в header
